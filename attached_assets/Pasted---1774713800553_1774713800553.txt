---

ДОПОЛНИ СУЩЕСТВУЮЩИЙ ПРОЕКТ.

СТРОГО:

* Вносить изменения ТОЛЬКО в указанные экраны и роли
* НЕ менять архитектуру приложения
* НЕ переносить логику между ролями

==================================================
=== КЛИЕНТ + ФРАНЧАЙЗИ ===
==========================

1. ОТОБРАЖЕНИЕ НАЛИЧИЯ ТОВАРА:

* Под КАЖДЫМ товаром везде (каталог, лента, товары у франчайзи) всегда отображать статус наличия

ЛОГИКА:

* если stock > 0 → текст "В НАЛИЧИИ"

* если stock == 0 → текст "ПРЕДЗАКАЗ"

* Это правило должно применяться:

  * в ленте клиента
  * в карточке товара
  * в разделе "Товары" у франчайзи

НЕ делать выборочное отображение — всегда показывать статус

2. ЛОГИКА ОСТАТКОВ (СИНХРОНИЗАЦИЯ):

* При оформлении заказа клиентом:
  → stock уменьшается на количество заказанных единиц

* При отмене заказа клиентом:
  → stock автоматически восстанавливается на количество отменённого заказа

* Обязательно учитывать количество товара в заказе (например 3 единицы → +3 при отмене)

* Обновление должно быть синхронным для:

  * клиента
  * франчайзи

==================================================
=== ФРАНЧАЙЗИ ===
=================

1. УДАЛЕНИЕ ТОВАРА:

* В режиме редактирования товара добавить кнопку:
  → "Удалить товар"

* При удалении:
  → товар удаляется полностью из базы
  → исчезает у клиента и из всех списков

==================================================
=== КЛИЕНТ ===
==============

1. УДАЛЕНИЕ ЗАКАЗА (UI ПОВЕДЕНИЕ):

* При нажатии "Удалить заказ":
  → показывать модальное окно подтверждения ВНУТРИ приложения (не браузерное)

ТЕКСТ МОДАЛЬНОГО ОКНА:
"Вы точно хотите удалить товар из заказов?"

* Кнопки:

  * Отмена
  * Подтвердить удаление

* Только после подтверждения заказ удаляется

НЕ использовать системные/браузерные окна подтверждения

==================================================
=== ВАЖНО ===
=============

* НЕ менять структуру проекта
* НЕ добавлять новые экраны без необходимости
* НЕ дублировать логику
* Только расширить существующие функции

После изменений:

* перечислить изменённые файлы
* кратко описать реализацию:

  * статусов наличия
  * восстановление stock
  * модального окна удаления
