Нужно доработать форму регистрации в веб-приложении (React + Vite), чтобы она соответствовала нашему предыдущему дизайну и логике.

ВАЖНО:
- не менять стек
- не упрощать UI
- не делать “базовую форму”
- сделать так, как было задумано в нашем проекте (premium / fashion стиль)

====================================
ЗАДАЧА
====================================

Переделать экран регистрации.

====================================
1. UI / ДИЗАЙН
====================================

Сделать форму в стиле:
- минимализм
- premium / fashion
- тёмная тема (чёрный фон)
- белый текст
- аккуратные поля ввода
- правильные отступы
- центрированный layout

Форма должна выглядеть как полноценный mobile/web экран, а не как стандартный HTML form.

Добавить:
- заголовок (например: "Создать аккаунт")
- подзаголовок (короткий текст)
- красивые input поля
- кнопка регистрации (primary)
- ссылка "Уже есть аккаунт? Войти"

====================================
2. ПОЛЯ ФОРМЫ
====================================

Обязательные поля:

- ФИО
- Email
- Пароль
- Повтор пароля
- Роль (селектор):
  - Клиент
  - Франчайзи
  - Производство

====================================
3. ВАЛИДАЦИЯ
====================================

Реализовать:

- все поля обязательны
- email валидный
- пароль не пустой
- пароль = повтор пароля
- роль выбрана

Ошибки:
- показывать под полем
- не алертом
- аккуратно встроено в UI

====================================
4. UX
====================================

- при фокусе — подсветка поля
- при ошибке — красная подсветка
- кнопка disabled если форма невалидна
- loading state при отправке
- после успешной регистрации:
  → редирект на логин

====================================
5. ЛОГИКА
====================================

- использовать существующий API / сервис регистрации
- не ломать текущую архитектуру
- сохранить совместимость с backend

====================================
6. ЧИСТОТА КОДА
====================================

- вынести форму в отдельный компонент
- использовать hooks (useState / useForm если есть)
- не писать всё в одном файле
- не делать дублирование логики

====================================
7. ЧТО НЕЛЬЗЯ
====================================

- не делать простой bootstrap-вид
- не оставлять дефолтные стили
- не игнорировать роли
- не делать заглушки вместо логики

====================================
РЕЗУЛЬТАТ
====================================

В конце:

1. Покажи:
- файл формы
- где подключена
- какие стили добавлены

2. Подтверди:
- форма работает
- валидация работает
- регистрация проходит

3. UI должен выглядеть как полноценный продукт, а не как прототип
