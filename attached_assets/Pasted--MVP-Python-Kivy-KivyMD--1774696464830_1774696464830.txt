Создай MVP мобильного приложения на Python с использованием Kivy (или KivyMD, если нужно для удобной стилизации), строго следуя техническому заданию ниже.

ВАЖНО:

* Дизайн приложения должен ПОЛНОСТЬЮ соответствовать изображениям, которые я отправлю позже. Используй их как основной источник для UI (layout, spacing, typography, цвета).
* Не используй стандартный Material Design или iOS UI — интерфейс должен выглядеть премиально, минималистично и кастомно.

=== ОСНОВНАЯ ЗАДАЧА ===
Разработать MVP "супераппа" для fashion-франшизы AVISHU с тремя ролями:

1. Клиент
2. Франчайзи
3. Производство (Цех)

Приложение должно менять интерфейс в зависимости от роли пользователя после входа.

=== ГЛАВНЫЙ СЦЕНАРИЙ (обязательно реализовать) ===

1. Клиент создает заказ
2. Франчайзи видит заказ в реальном времени и принимает его
3. Заказ появляется в интерфейсе производства
4. Производство завершает заказ
5. У клиента автоматически обновляется статус заказа ("Готов")

=== ТЕХНОЛОГИИ ===

* Python + Kivy / KivyMD (Frontend)
* Firebase (Firestore + Auth) или Supabase (Backend)
* Реалтайм обновления (Firestore listeners или аналог)

=== АРХИТЕКТУРА ===

* Один код приложения для всех ролей
* Роль пользователя хранится в базе (поле role)
* После логина происходит маршрутизация на нужный интерфейс

=== ФУНКЦИОНАЛ ===

1. ЭКРАН АВТОРИЗАЦИИ:

* Вход / регистрация
* Получение роли пользователя (client / franchise / production)

2. РОЛЬ: КЛИЕНТ

* Главный экран:

  * Баннер коллекции
  * Список товаров (минималистично)
* Карточка товара:

  * Если "в наличии" → кнопка "Купить"
  * Если "предзаказ" → выбор даты (календарь)
* Экран заказов:

  * Статусы: "Оформлен" → "Пошив" → "Готов"
  * Визуальный прогресс-бар

3. РОЛЬ: ФРАНЧАЙЗИ

* Дашборд:

  * Заглушки метрик (выручка, план)
* Список заказов:

  * Реалтайм обновление
  * Возможность менять статус заказа

4. РОЛЬ: ПРОИЗВОДСТВО

* Очень простой интерфейс (как для планшета)
* Список задач (заказы)
* Крупная кнопка "Завершить"
* При нажатии обновляется статус заказа

=== БАЗА ДАННЫХ (пример) ===

users:

* id
* email
* role (client / franchise / production)

orders:

* id
* user_id
* product_name
* status (created / in_progress / done)
* created_at

=== UI/UX ТРЕБОВАНИЯ ===

* Цвета: черный (#000000), белый (#FFFFFF), серый
* Без ярких цветов и градиентов
* Верхний регистр (UPPERCASE) для заголовков
* Минимум текста
* Много отступов (air spacing)
* Тонкие линии (1px)
* Прямые углы или очень маленькое скругление
* Ощущение "журнальной верстки"

=== ТРЕБОВАНИЯ К КОДУ ===

* Разделение экранов (ScreenManager)
* Чистая структура проекта (папки: screens, services, models)
* Отдельный модуль для работы с Firebase/Supabase
* Комментарии в коде

=== ДОПОЛНИТЕЛЬНО ===

* Добавь моковые данные товаров
* Сделай простую навигацию между экранами
* Подготовь проект для запуска через: python main.py

=== РЕЗУЛЬТАТ ===

* Полностью рабочий MVP
* Возможность показать сценарий на демо:
  Клиент → Франчайзи → Производство → Клиент
